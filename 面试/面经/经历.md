# 2024.2.21  招聘流程注意事项

以下内容总结于**鱼皮**哥的博客，主要讲的是作为求职者，在整个招聘流程中应该注意的事情！<br>

首先要了解招聘流程：发布岗位 => 候选人初筛 => 简历初筛 => 简历二筛 => 面试 => 发 offer <br>


## 打招呼 -- 进行候选人初筛

自己在软件上给挺多hr发了信息，可是回复率不高，于是去看了**鱼皮**的博客，发现打招呼也是满满的技巧。<br>

首先看看**鱼皮**博客上给的一张图：<br>![Alt text](image/%E6%89%93%E6%8B%9B%E5%91%BC.png)<br>

确实可以看出来不可以用默认模板，因为大家都是海投的，所以**要在这18-20个字的有限空间凸现自己的优势和自己跟公司的匹配度**。<br>

1. 尽量不要让千篇一律的客套话占用了这宝贵的位置，比如 “您好，我对您的 XX 岗位很感兴趣。” 面试官都不知道接下来怎么回复。
2. 尽量不要用问句，比如 “你好，可以聊聊么？” 面试官：聊啥？有啥好聊的？
3. 您好，我符合贵司XX、XX要求，项目：http://xxxx（实际上能让招聘方看到网址的前 2 个字母就够了）

同样，在自我介绍的时候，除了写基本的技能、经历之外，**尽量写一些与众不同的内容**。如果没有特别的点，那么可以尽量去匹配公司的要求，比如公司处于 “急招” 阶段的话，可以在介绍中补充说明：自己可以立即到岗。<br>

如下图：<br>![Alt text](image/%E8%87%AA%E6%88%91%E4%BB%8B%E7%BB%8D.png)<br>


## 简历初筛

同时，发过去的简历也要注意以下问题：<br>

1. 有的同学在简历上写到自己把项目上线了，但是根本无法访问。这算上线了还是没上？
2. 简历上很简单、很常用的技术名词都能写错。是因为不认真呢？还是真的不会？
3. 自我介绍（招呼语）中提到的分布式项目，却没有写在简历上。我本来还挺期待的
4. 项目直接照搬网上的项目，没有做任何的换皮和扩展。这就没办法跟其他同学拉开区分度
5. 工作经历只写了公司名称，没有写具体负责的工作内容。让我无法通过简历来了解到候选人的真实经历
6. 更想看项目中遇到的问题，并是如何解决或优化的。但实际上大量的简历写的都是项目的介绍，说这个项目多大，复杂度多高，运用了多少技术，最后就写几句自己参与什么什么模块儿。然后就没了！！！
7. 很多人都写自己熟练运用什么技术，但其实你会的，别人都会，也都写了，实在看不到什么优势。请具体一点儿。


## 简历二筛

主要**是在通讯工具上和求职者交**流，进一步确认一些信息，便于面试官筛选出真正符合要求的同学。<br>

在比较标准化的招聘平台上，这一轮的状态一般是 “通过初筛”、或者 “沟通中”，算是面试前的最后一道坎。**如果是大公司统一招聘的话，基本上是没有这个环节的，直接给你来一场笔试、来一场性格测试就能够看出真实水平了**。<br>

同时在这一轮筛选中，要多说几句来主动争取机会，这显然是加分的。<br>![Alt text](image/%E7%AE%80%E5%8E%86%E4%BA%8C%E7%AD%9B%E6%B2%9F%E9%80%9A.png)<br>

关于和招聘方沟通，**鱼皮**哥的几条建议：<br>

1. 只要招聘方和你沟通，大概率是对你有一定意向的。不管你是否想去这家公司，多跟他们聊聊总不会有坏处，说不等能了解一些行情；而如果你是真的想争取面试，那一定要表现出积极的态度，关注消息及时回复。
2. 招聘方单独和你沟通，大概率是已经看过你的简历了，在你和其他候选人之间犹豫。这种时候，要进行 有效沟通 ，多补充一些简历上没有提到的内容，尤其是能够增加项目真实性的内容。如果真的是你自己做的项目，可以把学习过程中的笔记、代码给面试官分享一下；如果不是你自己做的项目，也可以在短时间内快速了解一下，现在不是有 GPT 么？
3. 可以从自己和岗位的匹配度下手，多给自己增加一些优势。比如公司的业务和 AI 有关，那么可以提一下自己学过的 AI 知识和用过的 AI 工具，都是动动嘴皮子就能加分的机会。

如下面的图中这种认真争取机会的同学，无论水平能否竞争过其他同学，面试官或许都是愿意给一次面试机会的。<br>

从招聘方的角度出发，并不缺能干活的人，缺的是能把活干好、态度认真的同学。哪怕是简单的增删改查代码，不同的人来做效率和成果也会大相径庭。<br>


## 面试

**鱼皮**哥分享的几个建议：<br>

1. 我关注候选人的**表达能力**(这个恰恰是自己所欠缺的)，这点从候选人开场的自我介绍就能感觉到了。有的同学自我介绍可以把自己的学习过程、自己会的技术、自己的优点、自己的项目经历、自己的工作经历一气呵成地讲清楚，并且在最后强调一下自己最大的优势，引导面试官去提问。但有的同学呢，介绍自己都含糊不清，自我介绍说了一堆，都是什么自己家在哪，自己爱唱跳 RAP 篮球之类的，和岗位没有什么直接关系。面试官接下来都不知道怎么问了，不就只能去问八股文了么？
2. 我关注候选人的**项目实践 > 技术水平**。因为绝大多数公司（包括我们）要找的不是理论研究者，而是实际的开发者、能干活的人；再加上现在网上不缺技术教程，学会某个技术并不难。所以比起直接问那些枯燥的八股文，我更关注候选人 能否实际应用技术 ，而不是纸上谈兵。

    怎么知道他能否实际应用技术呢？那就看他做过的项目，写过的代码，参与过的实习，接触过的业务场景。<br>

    例如下面几个项目问题就可以看得出来对于这个项目的理解程度：<br>

    1. 请介绍一下你这个项目的业务流程？包含的主要模块以及自己负责的工作。
    2. 你在做这个项目中，遇到的最大的难点、或者解决过的最复杂的 Bug 是什么？
    3. 你的项目有实际上线么？使用哪种方式进行部署？

    这几个问题，基本上是包装项目的 “杀手锏”！<br>

    也一定要注意，自己写在简历上的知识点，一定要去准备相关的面试题！扩展的知识可以答不出来，但是自己简历上说自己会的内容没答出来是很减分的。<br>

3. 候选人的其他品质。只需要几分钟的对话，你大概就知道对面是个怎么样的人了，性格是急性子还是慢性子、逻辑思维是强还是差、态度是积极还是消极。比如有的同学在 1 小时的面试中没有 1 次打断过面试官的话，都是等听清楚问题后、思考片刻再回答，体现出的就是一种沟通能力和沉稳；但有的同学在没听完我的问题，感觉 “这题我会！” 然后就抢答了，或者在我问项目 A 的时候，聊着聊着自己就讲到项目 B 去了，要是我不打断，我估计他还能把车从上海开到西伯利亚去。**整个面试的时间是有限的，不是说你全程说的话多得分就更多，而是要精准回答面试官的问题、让面试官在有限的时间内足够了解你。**

面试完后还可以反问面试官一些问题，例如**我这场面试表现怎么样？**，能从有耐心的面试官口中得到一些真实的看法，便于你自己后续的改进。从面试官的角度，我也会觉得这位同学态度是比较积极的。<br>

但这种问题就不用问了：**我能否通过本轮面试？**<br>


# 2024.02.23  简历项目

**鱼皮**哥给的几点小建议，让自己的简历项目看起来更真实。<br>

1. 把**项目上线**，并且提供可直接访问的网址。
2. **项目开源**：如果是你自己学习时做的项目，最好能把项目开源、提供开源地址，能够让面试官点进去看你写过的代码，也是能大幅增加真实性的。
   1. 认真编写项目的介绍文档，把项目的演示截图、主要功能、架构设计、关键实现细节写清楚，让面试官感受到你的文档编写能力。
   2. 最好不要在本地写完整个项目后再整体提交、或者直接 fork 开源项目，而是应该在本地多次提交。
   3. 如果你是学习了网上的教程或开源项目，千万别不做修改就提交，我建议还是自己认真学习、重敲一遍。
3. **提供文档**：可以用云文档来编写对整个项目的介绍、自己完成的功能的截图等。如果是跟着网上的教程项目学习，你可以整理好自己学习项目时的笔记，记录自己在项目开发过程中的思考、过程和解决问题的方法，并且提供文档地址。
4. **强调个人职责**：精简对项目的描述，突出个人在项目中扮演的角色、贡献和具体的工作。<br>
   在描述项目时，不要只是简单地列举项目的功能和特点。相反，**尽量精简项目描述，重点突出你在项目中的个人职责、贡献和 具体工作**。要强调你在项目中的独特价值，而不仅仅是项目本身的价值。换句话说，不要给人的感觉是在 “吹某个项目很厉害”，要给人的感觉是 “你很厉害”，因为 “你很厉害”，所以造就了一个 “很厉害的项目”。

    可以使用**明确的动词和量化数据**来描述你的工作。比如：我带领了 5 人团队，通过 Git Flow 管理项目协同，成功地将项目交付在规定的时间内，并通过 SQL 优化和压测实现了 50% 的性能提升。
5. **增加区分度**：对项目的个性化定制（换皮改界面）、增加额外的扩展功能（一般至少 2 个）等，从而让你的项目与其他人的项目有明显的区别。
6. **补充细节**：皮哥说这个可以写在项目描述，我觉得指的应该是开源的readme文档/所提供的文档这类地方，而不是简历上。<br>
   比如项目的周期大概多久，完成某个功能所使用的技术细节（比如自定义线程池的参数设置）、项目的运行环境（比如服务器的配置）、项目优化的效果（具体的比例 %）、测试结果以及项目取得的成果。这些细节将为你的项目增添真实性，拉开和其他同学项目的区分度、同时体现出你的专业性。
7. 2024.02.21中的面试模块提到的三个问题，基本盘要能答出来。

掘金的一篇文章，教你如何介绍自己的项目经验。[链接](https://juejin.cn/post/7017732278509453348)<br>

# 2024.3.13 面了两家公司后的提问

问题如下：
1. 公司在面试的时候通常会先让做自我介绍，从你的角度来看，应该从哪些方面介绍自己好呢？
2. 面过的几家公司，在面试的时候都问过以后正式工作想找什么方向的，想留在那个城市。是不是应该说要面的这个工作的方向(我是这么回答的)，至于城市，应该说面试公司所在地吗？例如我几次面的公司都是南京的公司，那么我应该说南京吗？如果我不是这么回答的话，面试官会不会觉得辛辛苦苦培养的人居然跑了，而且会降低自己的面试成功率。
3. 一般面试都会有好几轮，在笔试或机试过后的技术面，面试官问我有什么想问的，我应该问他技术栈 工作流程和要我干什么是吗？谈薪应该留在hr面再谈是吧？如果方便的话，可以详细讲讲每一轮面试该问面试官关于工作的什么问题吗？
4. 去面的每家公司，都把自己说的比较厉害，有前途。那么自己应该从哪些方面甄别这家公司是否有前途，这个岗位是否真如面试官说得那么好。还有就是面试官都会说我来这个岗位肯定可以得到很好的培养，虽然你也在星球里说过很多次了这主要看领导，那么除了这个是不是就没办法判断能否得到好的培养了呢？
5. 我想工作的方向是嵌入式开发，但是现在实习只要是和C++开发的我都投了，主要是刷刷面试经验。我想问卡哥的是，如果真有一个学校附近的公司想要我去，但是和我秋招想面的岗位不一样的，我应该去吗？例如前几天面了个dpu测开实习生，技术面试官说很满意，马上和我约了hr面，要是到时候过了应该去吗？还有hr面谈薪的时候该如何谈？举个例子招聘信息上写的300-400，那我该该如何定？


答案：
1. 这里如果哪方面比较优秀 就重点多说几句<br>
   自我介绍 就是简历的 浓缩版， 3分钟左右，做完自我介绍就好。<br>
   可以大体按照时间线来介绍，本科，研究生，都学了什么技术，做了什么项目，参加了什么比赛，去了哪里实习，等等。<br>
   例如，我是XX，家在XX，本科就读于XXX，大一的时候参加了XX比赛，取了XX成绩，之后开始学习XXX技术，并尝试用XX技术来做一个项目， 同时我也 获得了XXX奖学金，平时喜欢研究XX技术，经常逛XX论坛，也经常自己写一些技术总结，经常看相关的书。 兴趣爱好是XXX。<br>
   去了XXX公司实习，做了XX事情，这段经历也让我学习了XXX 。 <br>
   研究生的方向是XXX，主要解决XX环境下的XXX问题。 但自己对XXX技术更感兴趣，所以现在选择了XX方向，等等。 <br>
2. 如果你只想拿offer，没有自己偏好的话，那就面那个 就说想做哪方面的工作， 所在城市在哪 就说自己想在哪个城市。 
3. 不用那么纠结，就了解一下部门情况就可以了，这都不是最重要的环节。
   1. 技术面反问：项目组技术方向，技术栈都有哪些，团队分工 。 例如自己表现的如何，后序学习建议， 一般 来说不要问了，面试官也不会说， 但有的面试官还真会给你点建议，这个分人。 
   2. HR反问：部门业务方向，加班情况（也表明一下 自己对加班一点都不反感，可以疯狂加班），然后再说说和 技术面面试官聊得很好， 夸一夸技术面试官技术很强，部门技术氛围好。 
4. “那么自己应该从哪些方面甄别这家公司是否有前途” 说实话，你很难甄别， 项目，公司业务，技术栈 大概了解这些，匹配就去，其他看缘分， 能不能得到培养，这个基本就是看缘分，你没法提前知道。  面试官都会说 自己部门有培养
5. 这个要问你自己吧，如果你找不到实习的话，看你是否要坚持自己的方向，还是转测试。或者你拖一拖入职时间，自己在找一种。实习薪资都是 hr直接告诉你，基本不用谈。

所以自己的自我介绍大致可以遵循以下模板：<br>
我是 ** ，** 人，是 ** 的 ** 届毕业生。本科就读于 **** 大学，在本科期间连续两年获得校级奖学金。<br>
研究生就读于 ** ，方向是 Fl 和 KD，主要是解决分布式训练中的隐私泄露和模型压缩问题。但是自己对 开发方面的 技术更感兴趣，所以选择了现在这个方向。<br>
并且在研究生期间学习了关于开发方面的知识，学习的过程中会形成自己的技术总结和思考。并且还用这些知识做了简历上的项目。这个项目 ......(之后开始介绍下项目)<br>
兴趣爱好是篮球、健身，平常喜欢研究健身、功能性训练相关的书籍，也会根据书上的理论和自己的实践经历写相关的训练计划、技术总结。<br>
