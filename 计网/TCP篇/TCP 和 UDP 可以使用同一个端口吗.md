# TCP 和 UDP 可以同时绑定相同的端口吗
1. 首先介绍了TCP和UDP网络编程，感觉学的东西连起来了
2. 结论是可以，因为 TCP/UDP 各自的端口号相互独立


# 多个 TCP 服务进程可以绑定同一个端口吗
1. 结论：如果两个 TCP 服务进程同时绑定的 IP 地址和端口都相同，那么执行 bind() 时候就会出错，错误是“Address already in use”。注意IP地址为0.0.0.0的特殊情况
2. 重启 TCP 服务进程时，为什么会有“Address in use”的报错信息？如何避免？


# 客户端的端口可以重复使用吗
还是从四元组理解：内核是通过四元组信息来定位一个 TCP 连接的，并不会因为客户端的端口号相同，而导致连接冲突的问题。
