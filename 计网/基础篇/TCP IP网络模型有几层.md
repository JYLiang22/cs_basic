# 前言
首先提出了为什么要有TCP(传输控制协议 (Transmission Control Protocol))/IP(网际协议Internet Protoco)网络模型这个问题？
1. 对于同一台设备上的进程间通信，可以使用管道、消息队列、共享内存、信号等方式
2. 对于不同设备上的进程间通信，就需要网络通信，而设备是多样性的，所以要兼容多种多样的设备，就协商出了一套通用的网络协议

[为什么网络需要分层](https://www.zhihu.com/question/26424507)


# 应用层
1. 我们电脑或手机使用的应用软件都是在应用层实现
2. 应用层只需要专注于为用户提供应用功能，比如 HTTP、FTP、Telnet、DNS、SMTP等；而不用去关心数据是如何传输的。举例子寄快递
3. 应用层是工作在操作系统中的用户态，传输层及以下则工作在内核态


# 传输层
1. 存在两个协议TCP、UDP(功能、区别)
2. 要传输的数据包大小 > MSS(maximum segment size)，分块传输
3. 使用 **端口** 区分设备上要进行数据传输的应用


# 网络层
1. 消除对于传输层的误解：它不负责将数据从一个设备传输到另一个设备，这个功能由网络层负责
2. IP报文长度 > MTU(maximum transmission unit)将会分片，其中MTU由传输的数据包、TCP头部和IP头部组成
3. 网络层负责将数据从一个设备传输到另一个设备，世界上那么多设备，又该如何找到对方呢？因此，网络层需要有区分设备的编号： **IP地址**
4. IP地址寻址：网络号(区分子网)和主机号(标识同一子网下不同主机)
5. IP地址 & 子网掩码 = 网络地址；IP地址 & (!子网掩码) = 主机地址
6. IP地址路由：达到网络节点选择路径，看博客图
7. 寻址像是导航，路由像是操作方向盘


# 网络接口层
1. 介绍了以太网：在 **局域网** 内，把附近的设备连接起来使它们之间可以进行通讯的技术
2. 以太网使用MAC头部判断网络包目的地址，通过ARP协议(地址解析协议，Address Resolution Protocol)获取对方的MAC地址
