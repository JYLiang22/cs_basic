# 为什么会出现

![Alt text](%E4%B8%BA%E4%BB%80%E4%B9%88%E5%87%BA%E7%8E%B0.png)<br>

如图所示，外观模式提供一个外观角色封装多个复杂的子系统，简化客户端与子系统之间的交互，降低系统的耦合度，方便客户端的使用。


# 结构

![Alt text](%E7%BB%93%E6%9E%84%E5%9B%BE.png)<br>

1. 外观角色(Facade)：可以知道多个相关子系统的功能，它将所有从客户端发来的请求委派给相应的子系统，传递给对应的子系统处理
2. 子系统角色(SubSystem)：一个类或多个类的结合，实现了子系统的具体功能

# 优缺点

优点：

1. 使得客户端不必关系子系统组件，减少了与子系统的交互，简化了客户端编程
2. 大大降低系统的耦合度
3. 子系统变化不需要修改客户端，只需要修改外观类即可
4. 子系统之间不会相互影响

缺点：

1. 如果要增删子系统，就需要修改外观类，违反开闭原则

适用场景：

1. 为访问一系列复杂的子系统提供一个统一、简单的接口
2. 客户端与多个子系统之间存在很大依赖，需要降低系统耦合度
