# 进程、线程基础知识


## 进程

通过读取文件的例子引入 **并发**，并介绍 **并发和并行** 的区别。<br>
进程的状态：七状态图、如何转换、为什么要出现挂起状态等。<br>
进程的控制结构：PCB、PCB包含什么信息、PCB如何组织。<br>
进程的控制：创建、终止、阻塞、唤醒过程，可以结合项目里面的代码理解。<br>
进程的上下文切换：为什么要出现、CPU上下文切换(进程/线程/中断上下文切换)、切换的是什么(用户/内核空间)、哪些场景发生(大多数都是调度、中断等场景)。<br>


## 线程

为什么使用线程？举一个例子说明了单进程、多进程的缺点，进而提出线程。<br>
什么是线程？共享和独占的分别是什么资源，有什么优缺点？<br>
进程和线程的区别，**为什么线程切换开销比进程少**。<br>
线程的上下文切换，具体看要切换的线程是不是属于同一个进程。<br>
线程的实现：三种实现方式；用户线程和内核线程的三种对应方式，多对一、一对一、多对多；各有什么优缺点<br>


## 调度

结合七状态图理解调度时机，结合调度原则理解常见调度算法。<br>
调度算法的优缺点及发展：<br>
先来先服务 -> 最短作业优先 -> 高响应比优先 -> 时间片轮转调度 -> 最高优先级 -> 多级反馈队列<br>
