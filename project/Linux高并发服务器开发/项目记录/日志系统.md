# 单例模式与阻塞队列的定义


## 单例模式

为什么要使用单例模式？<br>

使用单例模式创建日志系统，这样的话可以确保，当多线程操作时，只有一个日志系统，可以确保系统正确运行工作。<br>


### 懒汉模式

博客在懒汉模式为什么需要双检测那里描述得太简单了吧，应该详细点描述。<br>

```cpp
class single{
private:
  //私有静态指针变量指向唯一实例
  static single *p;
  //静态锁，是由于静态函数只能访问静态成员
  static pthread_mutex_t lock;

  //私有化构造函数
  single(){
    pthread_mutex_init(&lock, NULL);
  }
  ~single(){}

public:
  //公有静态方法获取实例
  static single* getinstance();
};

pthread_mutex_t single::lock;

single* single::p = NULL;
single* single::getinstance(){
    pthread_mutex_lock(&lock);
    if (NULL == p){
      p = new single;
    }
  return p;
}
```

上面代码就是只检测一次的情况，首先需要创建一个lock来加锁，而不能通过直接lock(p)，因为实例可能还没被创建。<br>

但这种方法的坏处是，每一次调用getinstance方法时都需要lock，会影响性能，所以出现了双重锁定版本的懒汉模式。<br>

```cpp
class single{
private:
  //私有静态指针变量指向唯一实例
  static single *p;
  //静态锁，是由于静态函数只能访问静态成员
  static pthread_mutex_t lock;

  //私有化构造函数
  single(){
    pthread_mutex_init(&lock, NULL);
  }
  ~single(){}

public:
  //公有静态方法获取实例
  static single* getinstance();
};

pthread_mutex_t single::lock;

single* single::p = NULL;
single* single::getinstance(){
  if(NULL == p){
    pthread_mutex_lock(&lock);
    if (NULL == p){
      p = new single;
    }
    pthread_mutex_unlock(&lock);
  }
  return p;
}
```

至于为什么要用双重锁定，但凡学过操作系统的锁机制，都比较容易想到。<br>


### 饿汉模式

饿汉模式不需要用锁，就可以实现线程安全。原因在于，在程序运行时就定义了对象，并对其初始化。之后，不管哪个线程调用成员函数getinstance()，都只不过是返回一个对象的指针而已。所以是线程安全的，不需要在获取实例的成员函数中加锁。<br>

```cpp
class single{
private:
  static single* p;
  single(){}
  ~single(){}

public:
  static single* getinstance();
};

single* single::p = new single();
single* single::getinstance(){
  return p;
}

//测试方法
int main(){

  single *p1 = single::getinstance();
  single *p2 = single::getinstance();

  if (p1 == p2)
    cout << "same" << endl;

  system("pause");
  return 0;
}
```
